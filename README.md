# **🤖 業務トラブル判定AIエージェント**

### **〜 現場の「カオス」を構造化し、初動の摩擦をゼロにする 〜**

## **🚀 プロジェクト・コンセプト**

### **「思考構造をPythonで記述する」**

本プロジェクトは、単なる自動応答システムではありません。現場から届く「粒度様々なSOS」を、設計者が定義した9つの思考構造（Schema）に強制的に落とし込み、役割を分担した複数のAIエージェントが「再現可能な意思決定」を代行するプラットフォームです。

## **🧠 コア・ロジック：現場の不条理を救う「物差し」**

AI任せにせず、設計者が定義した「現場の温度感」をスコアリングに反映しています。

### **📊 スコアリング・マトリクス**

| 判定項目 | 内容 | 加点 |
| :---- | :---- | :---- |
| **顧客ランク** | Enterprise(40) / Standard(20) / Free(0) | \+0〜40 |
| **データリスク** | DB破壊・整合性欠如の懸念がある場合 | **\+50** |
| **期限（圧）** | 解決期限まで60分を切っている場合 | \+30 |
| **頻発環境** | 同一IDで7日以内に3回以上の申告 | **\+20** |

### **🛡️ 独自実装：複合リスク補正 (Compound Risk Correction)**

**「重大なリスクがある」** かつ **「担当者に権限がない」** という、現場が最も孤立し心理的安全性が壊れやすい制約を検知した際、判定スコアを **自動で1.5倍に補正** します。これにより、組織的なバックアップを強制的に発動させ、迅速な解決へと繋げます。

## **🏗 アーキテクチャ：責任と役割の分離**

開発効率とメンテナンス性を高めるため、機能を独立した「エージェント」へ解体しています。

* **src/schema.py** 【器】: 現場の状況を定義する「思考の型」。  
* **agents/classifier.py** 【脳】: スコアリングと緊急度判定を担当。  
* **agents/planner.py** 【手】: 現場が今すぐやるべき「止血策」を立案。  
* **pipeline.py** 【道】: サーバー不要で全工程を完走させる実証環境。
